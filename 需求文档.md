# 智能待办网站 PRD（简洁版）

## 1. 产品概述
智能待办是一款以**自然语言输入为核心**的在线待办事项管理工具，旨在让用户以最低操作成本记录任务、查看任务与完成任务。产品强调“效率优先”，支持自然语言解析、智能提醒与结构化管理。

**一句话愿景**：更快记录、更少点击，让任务管理变得零负担。

**产品定位**：智能助理风的待办应用（偏效率流，参考 Notion AI / Things + LLM）。

**目标周期**：1周 MVP 可上线。

---

## 2. 产品目标与价值
| 目标 | 价值 |
|---|---|
| 以自然语言快速录入任务 | 降低记录成本，提高使用频次 |
| 智能解析时间/优先级/标签 | 自动结构化，降低重复操作 |
| 自动提醒与周报 | 减少遗忘，提高闭环能力 |
| 多终端可访问（Web 响应式） | 随时记录，随时查看 |

**MVP 成功判断标准：**
- 新增任务 ≤ 5 秒完成（含解析）
- 解析正确率 ≥ 80%
- 提醒功能可验证生效（邮件或 WebPush 至少一种）

---

## 3. 用户与使用场景
### 目标用户
- 有持续任务管理需求的人（白领、学生、自由职业者）
- 偏好聊天式/轻输入式工具，而非表格式操作

### 核心场景示例
| 场景 | 示例输入 |
|---|---|
| 临时任务快速记录 | “明早9点开会 准备汇报PPT” |
| 规划事项 | “每周一上午做周报” |
| 查询 | “这周我还有什么没做？” |
| 查看安排 | 打开日历或任务清单 |

---

## 4. 产品核心功能
### ✅ Must（MVP 必须有）
- 用户注册 / 登录
- 待办 CRUD：新建、编辑、完成、删除
- 自然语言录入任务（解析时间/提醒/优先级）
- 列表视图（默认按时间排序）
- 日历视图（最简周视图或月视图）
- 提醒通知（邮件 或 Web Push 任选一个）
- 响应式 Web UI

### ✨ Should（AI 亮点，至少 1 个）
- 智能周报（统计本周任务完成情况 + 建议）

### 🌱 Optional（可选，有时间再做）
- 标签管理 / 任务分组
- 看板视图（Trello 式）
- 语音输入
- 共享任务

---

## 5. 用户流程（核心——自然语言录入）
解析字段包括：时间 / 提醒时间 / 优先级 / 标签（可选）  
若解析不确定 → 系统给出候选方案 → 用户选择

---

## 6. 主要页面与交互
| 页面 | 关键交互点 |
|---|---|
| 首页（任务列表） | 顶部自然语言输入框 + 卡片式任务列表 |
| 日历页 | 用于查看任务分布 |
| 任务详情弹层 | 标题、描述、优先级、提醒设置 |
| 登录注册页 | 基本鉴权 |

默认交互原则：**一切从输入框开始**

---

## 7. 数据结构（ER 概览）
需要的核心数据表：
- `users`
- `todos`（含：title, description, due_at, remind_at, priority, status, nlu_raw）
- `reminders`（用于提醒触发）

主索引：todos(user_id, status, due_at)

---

## 8. 非功能性要求
- 简洁 UI，减少点击路径
- 操作可回退（Undo 完成或删除）
- 移动端优先（适配良好）
- 保障数据安全（鉴权 + https + 密码加密）

---

## 9. 验收标准（MVP）
- ✅ 任务可通过自然语言创建
- ✅ 系统可解析时间字段并正常提醒
- ✅ 列表和日历可查看所有任务
- ✅ 数据持久化正常
- ✅ UI 在 PC 和手机可顺畅使用

---

## 10. 项目节奏（7 天）
| 日期 | 目标 |
|---|---|
| Day1 | 框架 + 登录 |
| Day2 | CRUD + 列表 |
| Day3 | 自然语言解析 |
| Day4 | 日历视图 |
| Day5 | 提醒系统 |
| Day6 | 周报 + Fix |
| Day7 | 部署交付 |

---

## 结语
该 MVP 版本确保：**“记得快、看得清、不会忘”**，并为后续 AI 深化保留扩展空间（如智能建议、知识式任务管理、情境任务等）。
